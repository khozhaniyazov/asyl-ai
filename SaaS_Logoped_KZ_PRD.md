# 📄 Product Requirements Document (PRD): SaaS для логопедов-дефектологов в Казахстане
**Версия:** 1.0 (Release Candidate для MVP)
**Статус:** Утверждено к разработке
**Целевой рынок:** Республика Казахстан (B2B SaaS для частных специалистов и небольших кабинетов)

## 1. Executive Summary (Суть продукта)
Специализированная облачная платформа, объединяющая CRM, биллинг и электронные медицинские карты (EMR) с интеграцией ИИ. Продукт решает главную боль рынка — острую нехватку времени у специалистов из-за ручного ведения бумажной документации и администрирования клиентов в разрозненных мессенджерах.

**Ключевое ценностное предложение:** Логопед просто проводит урок с включенным микрофоном, а система сама формирует стандартизированный медицинский отчет (SOAP-ноту) и отправляет домашнее задание родителям в WhatsApp.

## 2. Архитектура и Технологический Стек
Проект использует гибридную, легковесную архитектуру для обеспечения максимальной гибкости на старте без раздувания серверных расходов.

### 2.1 Backend & AI Logic (The "Markdown-driven" Approach)
Вместо жесткого кодирования бизнес-логики ИИ в базу данных, используется модульный подход:
* **LLM Core:** **Gemini API**. Выбран за высокую скорость работы с текстом и наличие щедрых бесплатных лимитов (Free Tier), что критически важно для тестирования гипотез и минимизации костов на этапе MVP.
* **Конфигурация Агентов:** Системные промпты, структура SOAP-нот и параметры медицинских шаблонов хранятся в обычных `.md` (Markdown) файлах на сервере. Это позволяет на лету редактировать логику поведения ИИ без передеплоя приложения и сложных миграций базы данных.
* **Speech-to-Text:** Whisper API (или легковесные локальные модели) для транскрибации аудио с занятий.
* **Основной Backend-фреймворк:** Node.js (NestJS / Express) или Python (FastAPI).

### 2.2 Frontend
* **Стек:** React.js или Vue.js (SPA).
* **Адаптивность:** Mobile-first подход для интерфейса родителя (клиентский портал) и Tablet-first для кабинета логопеда (использование на планшетах во время приема).
* **Локализация:** Обязательная поддержка казахского и русского языков интерфейса (i18n).

### 2.3 Интеграции
* **Платежи:** Kaspi Pay API (генерация счетов и QR-кодов для оплаты).
* **Уведомления:** WhatsApp Business API (напоминания о сессиях, отправка домашних заданий).

## 3. Матрица Функционала MVP

### Модуль 1: CRM и Умное Расписание
* Интерактивный календарь (день/неделя/месяц) с цветовым кодированием статусов (запланировано, оплачено, отменено).
* Автоматическая отправка WhatsApp-напоминаний родителям за 24 часа до приема.
* База данных пациентов с историей посещений.

### Модуль 2: Финансы и Биллинг (Локализация под КЗ)
* Трекинг статуса оплат для каждого занятия.
* Генерация ссылки на оплату через Kaspi в один клик из карточки приема.
* Финансовый дашборд специалиста (доход за месяц, список должников).

### Модуль 3: EMR и AI-Ассистент (Killer Feature)
* Запись аудио прямо из браузера/приложения во время сессии.
* Автоматическая транскрибация речи (голос -> сырой текст).
* Маршрутизация текста в Gemini API через конфигурационный Markdown-промпт.
* Генерация структурированной SOAP-ноты.
* Извлечение блока "Домашнее задание" из ноты и авто-отправка родителю.

## 4. ИИ-Пайплайн и Системный Промпт
Файл конфигурации агента: `prompts/soap_generator.md`

**System Role:** Ты — высококвалифицированный клинический логопед-дефектолог. Твоя задача — проанализировать сырую транскрипцию сессии с пациентом и сформировать профессиональный медицинский отчет в формате SOAP (Subjective, Objective, Assessment, Plan).

**Входные данные (Инжектируются Backend'ом):**
- Диагноз пациента: `{{patient_diagnosis}}`
- Цели терапии из прошлой сессии: `{{previous_plan}}`
- Сырая транскрипция аудио: `{{raw_transcript}}`

**Правила формирования (Rules):**
1. **S (Subjective):** Жалобы родителей (если есть), настроение и вовлеченность ребенка.
2. **O (Objective):** Выполненные упражнения (моторика, дыхание), отрабатываемые звуки, применяемые инструменты.
3. **A (Assessment):** Оценка динамики. Что получилось, какие трудности сохраняются.
4. **P (Plan):** План на следующий урок и четкое, понятное домашнее задание.

**Output Format:**
Верни ответ строго в формате JSON:
{
  "subjective": "...",
  "objective": "...",
  "assessment": "...",
  "plan": "...",
  "homework_for_parents": "..."
}

## 5. User Flow (Основной сценарий)
1. **Запись:** Логопед создает слот `Appointment` -> Система генерирует ссылку `Kaspi Pay` -> Отправляет WhatsApp маме.
2. **Прием:** Статус меняется на `paid`. Логопед открывает карту `Patient` -> Нажимает "Record Audio".
3. **Магия:** Аудио -> Whisper (Text) -> LLM (Gemini) обрабатывает текст через `soap_generator.md`.
4. **Финал:** Логопед видит заполненный черновик в своей системе, правит (при необходимости) -> Нажимает "Сохранить" -> `homework_for_parents` улетает в WhatsApp родителям.
